Modelo Scrum para el Cálculo del Área de una Figura Geométrica

Roles Scrum

  • Product Owner: Define los requerimientos de la aplicación.
  • Scrum Master: Facilita el proceso Scrum y ayuda a eliminar impedimentos.
  • Equipo de Desarrollo: Implementa la lógica de cálculo y el diseño de la interfaz.

Product Backlog

Lista priorizada de funcionalidades y requisitos:

  1. Soporte para diferentes figuras geométricas.
  2. Interfaz de usuario.
  3. Validaciones de entrada.
  4. Mostrar resultados de manera precisa.
  5. Escribir pruebas para verificar cálculos.
  6. Documentación de la aplicación.

Sprint Planning

Sprint Goal: Desarrollar la funcionalidad básica de cálculo de áreas para triángulo, cuadrado y círculo.

  • Implementar lógica de cálculo del área para triángulo.
  • Implementar lógica de cálculo del área para cuadrado.
  • Implementar lógica de cálculo del área para círculo.
  • Diseñar una interfaz gráfica básica.
  • Crear validaciones para inputs.

Desarrollo durante el Sprint

Eventos clave:

  • Daily Scrum: Reunión diaria para revisar el progreso.
  • Scrum Review: Presentación de lo completado al final del sprint.
  • Scrum Retrospective: Revisión de lo que salió bien y lo que se puede mejorar.
